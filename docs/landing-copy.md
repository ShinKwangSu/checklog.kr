## 2. Problem — 현장에서 매일 반복되는 3가지

**섹션 제목**
아직도 이렇게 관리하고 계신가요?

**문제 1 — 점검 기록이 사라집니다**
종이·엑셀·카톡에 흩어진 점검 기록.
월말이면 현장 5곳의 기록을 모으는 데만 **반나절**이 걸리고,
"그날 점검했나요?"라는 질문에 답할 근거가 남지 않습니다.

**문제 2 — 현장 현황을 전화로 확인합니다**
어느 건물이 오늘 점검을 마쳤는지 알려면 **현장마다 전화**를 돌려야 합니다.
관리 현장이 10곳이면, 확인 전화만 하루 10통.

**문제 3 — 민원이 기록 없이 사라집니다**
전화·구두로 들어온 민원은 **10건 중 3~4건**이 처리 과정에서 누락됩니다.
누가 언제 접수했고 처리했는지 남지 않아, 같은 불만이 반복됩니다.

## 3. Services — checklog이 바꾸는 것

**섹션 제목**
설치도 로그인도 없이, 스캔 한 번으로

**서비스 1 — QR 점검으로 종이 점검표 대체**
시설마다 붙은 QR을 스캔하면, 점검한 사람·시각·항목이 그 자리에서 디지털로 기록됩니다.
매일·매주·매월 반복하던 점검표를 그대로 옮겨오고, 종이처럼 **찢어지거나 분실되지 않습니다.**
쌓인 이력은 자동 집계되어 그대로 남고, "그날 점검했나요?"에 **클릭 한 번으로** 답할 수 있습니다.
> 효과: 종이 보관·분실 리스크 **0**, 월말 이력 취합(현장별 종이 → 엑셀 재입력) **반나절 → 즉시**

**서비스 2 — 현장 대시보드**
오늘·이번 주·이번 달 점검 현황이 현장별로 자동 집계됩니다.
확인 전화 대신 화면 한 번으로, **현황 파악에 쓰던 하루 30분을 5초**로.
> 효과: 확인 전화 **월 200통 → 0통**, 월말 집계 시간 **반나절 → 즉시**

**서비스 3 — QR 민원 접수**
방문자가 QR로 사진과 함께 민원을 남기고, 접수 → 처리중 → 완료까지 상태로 추적됩니다.
구두 민원이 사라지니 **누락률이 30%대에서 0%로**.
> 효과: 민원 누락 **0건**, 처리 이력 **100% 기록**

## 5. Features — 기능소개 페이지

> 톤: 고객(현장 관리자)이 이해하는 용어만 사용. CRUD·소프트 딜리트·테넌트·쿼리 등 개발 용어 금지.

**현장별 워크스페이스**
업체 하나가 여러 건물·장소를 독립 워크스페이스로 관리합니다. 데이터는 워크스페이스 단위로 완전히 격리됩니다.
- 건물/장소별 워크스페이스 분리
- 지상·지하 층수 정의 후 3F·B1 형태로 자동 치환 표시
- 현장 간 데이터 완전 격리(서로 노출·공유 없음)

**시설 타입 · 시설 정보 관리**
화장실·회의실 등 공간 카테고리를 자유롭게 정의하고, 시설 정보를 층·타입과 연결해 등록합니다.
- 워크스페이스별 시설 관리
- 시설명·층수·타입·위치설명·비고 등록
- 층·타입 기준으로 원하는 시설을 빠르게 정렬·검색

**점검표 · 점검자 관리**
반복 주기가 있는 점검표 템플릿과 현장 점검자를 워크스페이스 단위로 운영합니다.
- 매일·매주·매월 반복 주기와 점검 항목 구성
- 점검자(담당자) 정보 관리
- 시설-점검표 다대다 연결

**QR 현장 점검**
시설에 부착된 QR을 스캔해 별도 로그인 없이 현장에서 바로 점검을 수행합니다.
- 시설별 QR 코드 발급
- 비로그인 현장 접근
- 점검 이력 자동 누적(오늘·이번주·이번달)

**민원 접수 · 처리**
방문자가 QR로 불편사항을 접수하고, 관리자는 시설별로 민원을 확인·처리합니다.
- 비로그인 방문자 민원 접수(사진 첨부)
- 시설별 민원 이력 확인
- 접수 → 처리중 → 완료 상태 관리

**데이터 격리 · 보안**
업체·현장 이중 분리와 로그인 기반 접근 제어로 각 업체의 데이터를 안전하게 보호합니다.
- 업체 + 현장 단위 데이터 분리
- 로그인 기반 접근 권한 관리
- 실수로 삭제해도 복구 가능
